Install Primeagent

Primeagent can be installed in multiple ways:

  • Primeagent Desktop (Recommended): Download and install the standalone desktop application for the least complicated setup experience. This option includes dependency management and facilitated upgrades.

  • Docker: Pull and run the Primeagent Docker image to start a Primeagent container and run Primeagent in isolation.

  • Python package: Install and run the Primeagent OSS Python package. This option offers more control over the environment, dependencies, and versioning.

  • Install from source: Use this option if you want to contribute to the Primeagent codebase or documentation.

Install and run Primeagent Desktop​

Primeagent Desktop is a desktop version of Primeagent that simplifies dependency management and upgrades. However, some features aren't available for Primeagent Desktop, such as the Shareable Playground and Voice Mode.

  • macOS
  • Windows

Primeagent Desktop requires macOS 13 or later.

  1. Navigate to Primeagent Desktop.
  2. Click Download Primeagent, enter your contact information, and then click Download.
  3. Mount and install the Primeagent application.
  4. When the installation completes, open the Primeagent application, and then create your first flow with the Quickstart.
warning

If you are upgrading Primeagent Desktop on Windows, don't use the in-app update feature to upgrade to Primeagent version 1.6.0. For more information, see Known issues for 1.6.0.

  1. Navigate to Primeagent Desktop.

  2. Click Download Primeagent, enter your contact information, and then click Download.

  3. Open the File Explorer, and then navigate to Downloads.

  4. Double-click the downloaded .msi file, and then use the install wizard to install Primeagent Desktop.

    tip

    Windows installations of Primeagent Desktop require a C++ compiler that may not be present on your system. If you receive a C++ Build Tools Required! error, follow the on-screen prompt to install Microsoft C++ Build Tools, or install Microsoft Visual Studio.

  5. When the installation completes, open the Primeagent application, and then create your first flow with the Quickstart.

For upgrade information, see the Release notes.

To manage dependencies in Primeagent Desktop, see Install custom dependencies in Primeagent Desktop.

Install and run Primeagent with Docker​

You can use the Primeagent Docker image to start a Primeagent container. For more information, see Deploy Primeagent on Docker.

  1. Install and start Docker.

  2. Pull the latest Primeagent Docker image and start it:


    _10
    docker run -p 7860:7860 khulnasoft/primeagent:latest

  3. To access Primeagent, navigate to http://localhost:7860/.

  4. Create your first flow with the Quickstart.

Install and run the Primeagent OSS Python package​

  1. Make sure you have the required dependencies and infrastructure:

    • Python
      • macOS and Linux: Version 3.10 to 3.13
      • Windows: Version 3.10 to 3.12
    • uv
    • Sufficient infrastructure:
      • Minimum: Dual-core CPU and 2 GB RAM
      • Recommended: Multi-core CPU and at least 4 GB RAM
    • Browser:
      • Google Chrome is recommended but not required

  2. Create a virtual environment with uv.

    Need help with virtual environments?

    Virtual environments ensure Primeagent is installed in an isolated, fresh environment. To create a new virtual environment, do the following.

    • Linux or macOS
    • Windows

    1. Navigate to where you want your virtual environment to be created, and then create it with uv:


      _10
      uv venv VENV_NAME

      Replace VENV_NAME with a name for your virtual environment.

    2. Start the virtual environment:


      _10
      source VENV_NAME/bin/activate

      Your shell's prompt changes to display that you're currently working in a virtual environment:


      _10
      (VENV_NAME) ➜ primeagent git:(main) ✗

    3. To deactivate the virtual environment and return to your regular shell, type deactivate.

      When activated, the virtual environment temporarily modifies your PATH variable to prioritize packages installed within the virtual environment. To avoid conflicts with other projects, it's a good idea to deactivate your virtual environment when you're done working in it.

      To delete the virtual environment, type rm -rf VENV_NAME. This completely removes the virtual environment directory and its contents.

    1. Navigate to where you want your virtual environment to be created, and create it with uv.


      _10
      uv venv VENV_NAME

      Replace VENV_NAME with a name for your virtual environment.

    2. Start the virtual environment:


      _10
      VENV_NAME\Scripts\activate

      Your shell's prompt changes to display that you're currently working in a virtual environment:


      _10
      (VENV_NAME) PS C:/users/username/primeagent-dir>

    3. To deactivate the virtual environment and return to your regular shell, type deactivate.

      When activated, the virtual environment temporarily modifies your PATH variable to prioritize packages installed within the virtual environment. To avoid conflicts with other projects, it's a good idea to deactivate your virtual environment when you're done working in it.

      To delete the virtual environment, type Remove-Item VENV_NAME. This completely removes the virtual environment directory and its contents.

  3. In your virtual environment, install Primeagent:


    _10
    uv pip install primeagent

    To install a specific version of the Primeagent package, add the required version to the command, such as uv pip install primeagent==1.4.22.

    Reinstall or upgrade Primeagent

    To reinstall Primeagent and all of its dependencies, run uv pip install primeagent --force-reinstall.

    To upgrade Primeagent to the latest version, run uv pip install primeagent -U. However, the Primeagent team recommends taking steps to backup your existing installation before you upgrade Primeagent. For more information, see Prepare to upgrade.

  4. Start Primeagent:


    _10
    uv run primeagent run

    It can take a few minutes for Primeagent to start.

  5. To confirm that a local Primeagent instance is running, navigate to the default Primeagent URL http://127.0.0.1:7860.

  6. Create your first flow with the Quickstart.

For upgrade information, see the Release notes.

For information about optional dependency groups and support for custom dependencies to extend Primeagent OSS functionality, see Install custom dependencies.
