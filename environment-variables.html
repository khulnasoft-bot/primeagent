Environment variables

In general, environment variables, like PRIMEAGENT_PORT or PRIMEAGENT_LOG_LEVEL, configure how Primeagent runs. These are broad settings that apply to your entire Primeagent deployment.

In contrast, global variables are user-defined values stored in Primeagent's database for use in flows, such as OPENAI_API_KEY. Primeagent can also source global variables from environment variables. For more information, see Primeagent global variables.

Configure environment variables for Primeagent OSS​

Primeagent recognizes supported environment variables from the following sources:

  • Environment variables set in your terminal.
  • Environment variables imported from a .env file when starting Primeagent.
  • Environment variables set with the Primeagent CLI, including the --env-file option and direct options, such as --port.

You can choose to use one or more of these sources.

Precedence​

If the same environment variable is set in multiple places, the following hierarchy applies:

  1. Primeagent CLI options override all other sources.
  2. The .env file overrides system environment variables.
  3. System environment variables are used only if not set elsewhere. When running a Primeagent Docker image, the -e flag can be used to set additional system environment variables.

For example:

  • If you set PRIMEAGENT_PORT=8080 in your system environment and PRIMEAGENT_PORT=7860 in .env, Primeagent uses 7860 from .env.
  • If you use the Primeagent CLI to run primeagent run --env-file .env --port 9000, and you set PRIMEAGENT_PORT=7860 in .env, then Primeagent uses 9000 from the CLI option.

Run the following commands to set environment variables for your current terminal session:

  • Linux or macOS
  • Windows
  • Docker

When you start Primeagent, it looks for environment variables that you've set in your terminal. If it detects a supported environment variable, then it automatically adopts the specified value, subject to precedence rules.

Set environment variables for Primeagent Desktop​

Environment variables set in your terminal aren't automatically available to GUI-based applications like Primeagent Desktop when you launch them from the Windows or macOS GUI.

For Windows, this means any GUI-based app launched from the Start menu, desktop shortcuts, or Windows Explorer.

For macOS, this means any GUI-based app launched from Finder, Spotlight, Launchpad, or the Dock.

To set environment variables for Primeagent Desktop, you need to use specific commands or files, depending on your OS.

  • macOS
  • Window System Properties
  • Powershell

Primeagent Desktop for macOS cannot automatically use variables set in your terminal, such as those in.zshrc or .bash_profile, when launched from the macOS GUI.

To make environment variables available to GUI apps on macOS, you need to use launchctl with a plist file:

  1. Create the LaunchAgents directory if it doesn't exist:


    _10
    mkdir -p ~/Library/LaunchAgents

  2. In the LaunchAgents directory, create a .plist file called dev.primeagent.env.

  3. Add the following content to dev.primeagent.env.plist, and then add, change, or remove Primeagent environment variables as needed for your configuration.

    This example sets multiple environmental variables for all GUI apps launched from the macOS GUI.


    _21
    <?xml version="1.0" encoding="UTF-8"?>
    _21
    <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
    _21
    <plist version="1.0">
    _21
    <dict>
    _21
    <key>Label</key>
    _21
    <string>dev.primeagent.env</string>
    _21
    <key>ProgramArguments</key>
    _21
    <array>
    _21
    <string>/bin/sh</string>
    _21
    <string>-c</string>
    _21
    <string>
    _21
    launchctl setenv PRIMEAGENT_CONFIG_DIR /Users/your_user/custom/config ;
    _21
    launchctl setenv PRIMEAGENT_PORT 7860 ;
    _21
    launchctl setenv PRIMEAGENT_HOST localhost ;
    _21
    launchctl setenv ARIZE_API_KEY ak-...
    _21
    </string>
    _21
    </array>
    _21
    <key>RunAtLoad</key>
    _21
    <true/>
    _21
    </dict>
    _21
    </plist>

  4. Load the file with launchctl:


    _10
    launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/dev.primeagent.env.plist

Primeagent Desktop for Windows cannot automatically use variables set in your terminal, such as those defined with set in cmd or $env:VAR=... in PowerShell, when launched from the Windows GUI.

To make environment variables available to the Primeagent Desktop app, you must set them at the user or system level using the System Properties interface or the Terminal.

To set environment variables using the System Properties interface, do the following:

  1. Press Win + R, enter SystemPropertiesAdvanced, and then press Enter.

  2. Click Environment Variables.

  3. Under User variables, click New.

    tip

    To apply the setting to all users, select System variables.

  4. Enter the name of the Primeagent variable you want to set, such as PRIMEAGENT_CONFIG_DIR, and the desired value, such as C:\Users\your_user\.primeagent_config.

  5. Click OK to save the variable.

  6. Repeat until you have set all necessary Primeagent environment variables.

  7. Launch or restart Primeagent Desktop to apply the environment variables.

To define environment variables for Windows using PowerShell, do the following:

  1. Enter the name of the Primeagent variable you want to set, such as PRIMEAGENT_CONFIG_DIR, and the desired value, such as C:\Users\your_user\.primeagent_config.

    To set an environment variable for the current user:


    _10
    [System.Environment]::SetEnvironmentVariable("PRIMEAGENT_CONFIG_DIR", "C:\Users\your_user\.primeagent_config", "User")

    To set an environment variable for all users (you must have Administrator priveleges):


    _10
    [System.Environment]::SetEnvironmentVariable("PRIMEAGENT_CONFIG_DIR", "C:\Primeagent\Config", "Machine")

  2. Repeat until you have set all necessary Primeagent environment variables.

  3. Launch or restart Primeagent Desktop to apply the environment variables.

PRIMEAGENT_OPEN_BROWSERBooleanFalseOpen the system web browser on startup.
PRIMEAGENT_HEALTH_CHECK_MAX_RETRIESInteger5Set the maximum number of retries for Primeagent's server status health checks.
PRIMEAGENT_WORKERSInteger1Number of worker processes.
PRIMEAGENT_WORKER_TIMEOUTInteger300Worker timeout in seconds.
PRIMEAGENT_SSL_CERT_FILEStringNot setPath to the SSL certificate file for enabling HTTPS on the Primeagent web server. This is separate from database SSL connections.
PRIMEAGENT_SSL_KEY_FILEStringNot setPath to the SSL key file for enabling HTTPS on the Primeagent web server. This is separate from database SSL connections.
PRIMEAGENT_DEACTIVATE_TRACINGBooleanFalseDeactivate tracing functionality.
PRIMEAGENT_CELERY_ENABLEDBooleanFalseEnable Celery for distributed task processing.
