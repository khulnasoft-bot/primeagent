In general, environment variables, like `PRIMEAGENT_PORT` or `PRIMEAGENT_LOG_LEVEL`, configure how Primeagent runs.
These are broad settings that apply to your entire Primeagent deployment.

In contrast, global variables are user-defined values stored in Primeagent's database for use in flows, such as `OPENAI_API_KEY`.
Primeagent can also source global variables from environment variables.
For more information, see [Primeagent global variables](/configuration-global-variables).

## Configure environment variables for Primeagent OSS

Primeagent recognizes [supported environment variables](#supported-variables) from the following sources:

- Environment variables set in your terminal.
- Environment variables imported from a `.env` file when starting Primeagent.
- Environment variables set with the [Primeagent CLI](./configuration-cli), including the `--env-file` option and direct options, such as `--port`.

You can choose to use one or more of these sources.

### Precedence {#precedence}

If the same environment variable is set in multiple places, the following hierarchy applies:

1. Primeagent CLI options override all other sources.
2. The `.env` file overrides system environment variables.
3. System environment variables are used only if not set elsewhere.
When running a Primeagent Docker image, the `-e` flag can be used to set additional system environment variables.

For example:

* If you set `PRIMEAGENT_PORT=8080` in your system environment and `PRIMEAGENT_PORT=7860` in `.env`, Primeagent uses `7860` from `.env`.
* If you use the Primeagent CLI to run `primeagent run --env-file .env --port 9000`, and you set `PRIMEAGENT_PORT=7860` in `.env`, then Primeagent uses `9000` from the CLI option.

### Set environment variables in your terminal {#configure-variables-terminal}

Run the following commands to set environment variables for your current terminal session:

<Tabs>
<TabItem value="linux-macos" label="Linux or macOS" default>

```bash
export VARIABLE_NAME='VALUE'
```

</TabItem>
<TabItem value="windows" label="Windows">

```
set VARIABLE_NAME='VALUE'
```

</TabItem>
<TabItem value="docker" label="Docker">

```bash
docker run -it --rm \
    -p 7860:7860 \
    -e VARIABLE_NAME='VALUE' \
    khulnasoft/primeagent:latest
```

</TabItem>
</Tabs>

When you start Primeagent, it looks for environment variables that you've set in your terminal.
If it detects a supported environment variable, then it automatically adopts the specified value, subject to [precedence rules](#precedence).

### Import environment variables from a .env file {#configure-variables-env-file}

1. If Primeagent is running, quit Primeagent.

2. Create a `.env` file, and then open it in your preferred editor.

3. Define [Primeagent environment variables](#supported-variables) in the `.env` file.

    <details>
    <summary>Example: .env</summary>

    ```text
    DO_NOT_TRACK=True
    PRIMEAGENT_AUTO_LOGIN=False
    PRIMEAGENT_AUTO_SAVING=True
    PRIMEAGENT_AUTO_SAVING_INTERVAL=1000
    PRIMEAGENT_BACKEND_ONLY=False
    PRIMEAGENT_BUNDLE_URLS=["https://github.com/user/repo/commit/hash"]
    PRIMEAGENT_CACHE_TYPE=async
    PRIMEAGENT_COMPONENTS_PATH=/path/to/components/
    PRIMEAGENT_CONFIG_DIR=/path/to/config/
    PRIMEAGENT_DATABASE_URL=postgresql://user:password@localhost:5432/primeagent
    PRIMEAGENT_DEV=False
    PRIMEAGENT_FALLBACK_TO_ENV_VAR=False
    PRIMEAGENT_HEALTH_CHECK_MAX_RETRIES=5
    PRIMEAGENT_HOST=localhost
    PRIMEAGENT_LANGCHAIN_CACHE=InMemoryCache
    PRIMEAGENT_MAX_FILE_SIZE_UPLOAD=10000
    PRIMEAGENT_MAX_ITEMS_LENGTH=100
    PRIMEAGENT_MAX_TEXT_LENGTH=1000
    PRIMEAGENT_LOG_LEVEL=error
    PRIMEAGENT_OPEN_BROWSER=False
    PRIMEAGENT_PORT=7860
    PRIMEAGENT_REMOVE_API_KEYS=False
    PRIMEAGENT_SAVE_DB_IN_CONFIG_DIR=True
    PRIMEAGENT_SECRET_KEY=somesecretkey
    PRIMEAGENT_STORE_ENVIRONMENT_VARIABLES=True
    PRIMEAGENT_SUPERUSER=adminuser
    PRIMEAGENT_SUPERUSER_PASSWORD=adminpass
    PRIMEAGENT_WORKER_TIMEOUT=60000
    PRIMEAGENT_WORKERS=3
    ```

    For additional examples, see [`.env.example`](https://github.com/khulnasoft/primeagent/blob/main/.env.example) in the Primeagent repository.

    </details>

4. Save and close `.env`.

5. Start Primeagent with your `.env` file:

    <Tabs>
    <TabItem value="local" label="Local" default>

    ```bash
    uv run primeagent run --env-file .env
    ```

    </TabItem>
    <TabItem value="docker" label="Docker">

    ```bash
    docker run -it --rm \
        -p 7860:7860 \
        --env-file .env \
        khulnasoft/primeagent:latest
    ```

    </TabItem>
    </Tabs>

    If your `.env` file isn't in the same directory, provide the path to your `.env` file.

On startup, Primeagent imports the environment variables from your `.env` file, as well as any others that you set in your terminal, and then adopts their specified values.

## Set environment variables for Primeagent Desktop

Environment variables set in your terminal aren't automatically available to GUI-based applications like Primeagent Desktop when you launch them from the Windows or macOS GUI.

For Windows, this means any GUI-based app launched from the Start menu, desktop shortcuts, or Windows Explorer.

For macOS, this means any GUI-based app launched from Finder, Spotlight, Launchpad, or the Dock.

To set environment variables for Primeagent Desktop, you need to use specific commands or files, depending on your OS.

<Tabs>
<TabItem value="macos" label="macOS" default>

Primeagent Desktop for macOS cannot automatically use variables set in your terminal, such as those in`.zshrc` or `.bash_profile`, when launched from the macOS GUI.

To make environment variables available to GUI apps on macOS, you need to use `launchctl` with a `plist` file:

1. Create the `LaunchAgents` directory if it doesn't exist:

    ```bash
    mkdir -p ~/Library/LaunchAgents
    ```

2. In the `LaunchAgents` directory, create a `.plist` file called `dev.primeagent.env`.

3. Add the following content to `dev.primeagent.env.plist`, and then add, change, or remove Primeagent environment variables as needed for your configuration.

    This example sets multiple environmental variables for all GUI apps launched from the macOS GUI.

    ```xml
     <?xml version="1.0" encoding="UTF-8"?>
     <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
     <plist version="1.0">
     <dict>
       <key>Label</key>
       <string>dev.primeagent.env</string>
       <key>ProgramArguments</key>
       <array>
         <string>/bin/sh</string>
         <string>-c</string>
          <string>
            launchctl setenv PRIMEAGENT_CONFIG_DIR /Users/your_user/custom/config &&
            launchctl setenv PRIMEAGENT_PORT 7860 &&
            launchctl setenv PRIMEAGENT_HOST localhost &&
            launchctl setenv ARIZE_API_KEY ak-...
          </string>
       </array>
       <key>RunAtLoad</key>
       <true/>
     </dict>
     </plist>
    ```
4. Load the file with `launchctl`:

    ```bash
    launchctl load ~/Library/LaunchAgents/dev.primeagent.env.plist
    ```

</TabItem>
<TabItem value="windows1" label="Window System Properties">

Primeagent Desktop for Windows cannot automatically use variables set in your terminal, such as those defined with `set` in `cmd` or `$env:VAR=...` in PowerShell, when launched from the Windows GUI.

To make environment variables available to the Primeagent Desktop app, you must set them at the user or system level using the **System Properties** interface or the Terminal.

To set environment variables using the System Properties interface, do the following:

1. Press <kbd>Win + R</kbd>, enter `SystemPropertiesAdvanced`, and then press <kbd>Enter</kbd>.
2. Click **Environment Variables**.
3. Under **User variables**, click **New**.

    :::tip
    To apply the setting to all users, select **System variables**.
    :::

4. Enter the name of the Primeagent variable you want to set, such as `PRIMEAGENT_CONFIG_DIR`, and the desired value, such as `C:\Users\your_user\.primeagent_config`.
5. Click **OK** to save the variable.
6. Repeat until you have set all necessary Primeagent environment variables.
7. Launch or restart Primeagent Desktop to apply the environment variables.

</TabItem>
<TabItem value="windows2" label="Powershell">

To define environment variables for Windows using PowerShell, do the following:

1. Enter the name of the Primeagent variable you want to set, such as `PRIMEAGENT_CONFIG_DIR`, and the desired value, such as `C:\Users\your_user\.primeagent_config`.

    To set an environment variable for the current user:
    ```powershell
    [System.Environment]::SetEnvironmentVariable("PRIMEAGENT_CONFIG_DIR", "C:\Users\your_user\.primeagent_config", "User")
    ```

   To set an environment variable for all users (you must have Administrator privileges):
   ```powershell
   [System.Environment]::SetEnvironmentVariable("PRIMEAGENT_CONFIG_DIR", "C:\Primeagent\Config", "Machine")
   ```

2. Repeat until you have set all necessary Primeagent environment variables.
3. Launch or restart Primeagent Desktop to apply the environment variables.

</TabItem>
</Tabs>

### Server

The following environment variables set base Primeagent server configuration, such as where the server is hosted, required files for SSL encryption, and the deployment type (frontend and backend, backend-only, development mode).

| Variable | Format | Default | Description |
|----------|--------|---------|-------------|
| `PRIMEAGENT_HOST` | String | `localhost` | The host on which the Primeagent server will run. |
| `PRIMEAGENT_PORT` | Integer | `7860` | The port on which the Primeagent server runs. The server automatically selects a free port if the specified port is in use. |
| `PRIMEAGENT_BACKEND_ONLY` | Boolean | `False` | Run only the Primeagent backend service (no frontend). |
| `PRIMEAGENT_DEV` | Boolean | `False` | Whether to run Primeagent in development mode (may contain bugs). |
| `PRIMEAGENT_OPEN_BROWSER` | Boolean | `False` | Open the system web browser on startup. |
| `PRIMEAGENT_HEALTH_CHECK_MAX_RETRIES` | Integer | `5` | Set the maximum number of retries for Primeagent's server status health checks. |
| `PRIMEAGENT_WORKERS` | Integer | `1` | Number of worker processes. |
| `PRIMEAGENT_WORKER_TIMEOUT` | Integer | `300` | Worker timeout in seconds. |
| `PRIMEAGENT_SSL_CERT_FILE` | String | Not set | Path to the SSL certificate file for enabling HTTPS on the Primeagent web server. This is separate from [database SSL connections](/configuration-custom-database#connect-primeagent-to-a-local-postgresql-database). |
| `PRIMEAGENT_SSL_KEY_FILE` | String | Not set | Path to the SSL key file for enabling HTTPS on the Primeagent web server. This is separate from [database SSL connections](/configuration-custom-database#connect-primeagent-to-a-local-postgresql-database). |
| `PRIMEAGENT_DEACTIVATE_TRACING` | Boolean | `False` | Deactivate tracing functionality. |
| `PRIMEAGENT_CELERY_ENABLED` | Boolean | `False` | Enable Celery for distributed task processing. |

For more information about deploying Primeagent servers, see [Primeagent deployment overview](/deployment-overview).

### Visual editor and Playground behavior

| Variable | Format | Default | Description |
|----------|--------|---------|-------------|
| `PRIMEAGENT_AUTO_SAVING` | Boolean | `True` | Whether to automatically save flows. |
| `PRIMEAGENT_AUTO_SAVING_INTERVAL` | Integer | `1000` | Set the auto-save interval in milliseconds if `PRIMEAGENT_AUTO_SAVING=True`. |
| `PRIMEAGENT_BUNDLE_URLS` | List[String] | `[]` | A list of URLs from which to load custom bundles. Supports GitHub URLs. If `PRIMEAGENT_AUTO_LOGIN=True`, flows from these bundles are loaded into the database. |
| `PRIMEAGENT_COMPONENTS_PATH` | String | Not set | Path to a directory containing custom components. Typically used if you have local custom components or you are building a Docker image with custom components. |
| `PRIMEAGENT_LOAD_FLOWS_PATH` | String | Not set | Path to a directory containing flow JSON files to be loaded on startup. Typically used when creating a Docker image with prepackaged flows. Requires `PRIMEAGENT_AUTO_LOGIN=True`. |
| `PRIMEAGENT_CREATE_STARTER_PROJECTS` | Boolean | `True` | Whether to create templates during initialization. If `false`, Primeagent doesn't create templates, and `PRIMEAGENT_UPDATE_STARTER_PROJECTS` is treated as `false`. |
| `PRIMEAGENT_UPDATE_STARTER_PROJECTS` | Boolean | `True` | Whether to update templates with the latest component versions when initializing after an upgrade. |
| `PRIMEAGENT_LAZY_LOAD_COMPONENTS` | Boolean | `False` | If `true`, Primeagent only partially loads components at startup and fully loads them on demand. This significantly reduces startup time but can cause a slight delay when a component is first used. |
| `PRIMEAGENT_EVENT_DELIVERY` | String | `streaming` | How to deliver build events to the frontend: `polling`, `streaming` or `direct`. |
| `PRIMEAGENT_FRONTEND_PATH` | String | `./frontend` | Path to the frontend directory containing build files. For development purposes only when you need to serve specific frontend code. |
| `PRIMEAGENT_MAX_ITEMS_LENGTH` | Integer | `100` | Maximum number of items to store and display in the visual editor. Lists longer than this will be truncated when displayed in the visual editor. Doesn't affect outputs or data passed between components. |
| `PRIMEAGENT_MAX_TEXT_LENGTH` | Integer | `1000` | Maximum number of characters to store and display in the visual editor. Responses longer than this will be truncated when displayed in the visual editor. Doesn't truncate outputs or responses passed between components. |
| `PRIMEAGENT_MAX_TRANSACTIONS_TO_KEEP` | Integer | `3000` | Maximum number of flow transaction events to keep in the database. |
| `PRIMEAGENT_MAX_VERTEX_BUILDS_TO_KEEP` | Integer | `3000` | Maximum number of vertex builds to keep in the database. Relates to [Playground](/concepts-playground) functionality. |
| `PRIMEAGENT_MAX_VERTEX_BUILDS_PER_VERTEX` | Integer | `2` | Maximum number of builds to keep per vertex. Older builds are deleted. Relates to [Playground](/concepts-playground) functionality. |
| `PRIMEAGENT_PUBLIC_FLOW_CLEANUP_INTERVAL` | Integer | `3600` | The interval in seconds at which data for [shared Playground](/concepts-playground#share-a-flows-playground) flows are cleaned up. Default: 3600 seconds (1 hour). Minimum: 600 seconds (10 minutes). |
| `PRIMEAGENT_PUBLIC_FLOW_EXPIRATION` | Integer | `86400` | The time in seconds after which a [shared Playground](/concepts-playground#share-a-flows-playground) flow is considered expired and eligible for cleanup. Default: 86400 seconds (24 hours). Minimum: 600 seconds (10 minutes). |
