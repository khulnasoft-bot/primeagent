Primeagent CLI

The Primeagent command line interface is the main interface for managing and running the Primeagent server.

The Primeagent CLI is automatically installed when you install the Primeagent package. It isn't available for Primeagent Desktop.

How to use the CLI​

The Primeagent CLI can be invoked in several ways, depending on your installation method and environment.

The recommended approach is to run the CLI with uv run from within a virtual environment where Primeagent is installed.

For example, to start Primeagent on the default port, run the following command:


_10
uv run primeagent run

If Primeagent is installed globally or added to your PATH, you can execute the CLI directly with primeagent.


_10
primeagent run

Precedence​

Primeagent CLI options override the values of environment variables set in your terminal or primary .env file.

For example, if you have PRIMEAGENT_PORT=7860 defined as an environment variable, and you run the CLI with --port 7880, then Primeagent sets the port to 7880 because the CLI option overrides the environment variable.

This also applies to Boolean environment variables. For example, if you set PRIMEAGENT_REMOVE_API_KEYS=True in your .env file, you can change it to False at runtime by running the CLI with --no-remove-api-keys.

Primeagent CLI options​

All Primeagent CLI commands support options that modify the command's behavior or set environment variables.

To set values for options, you can use either of the following syntax styles:

  • --option value
  • --option=value

Values with spaces must be surrounded by quotation marks:

  • --option 'Value with Spaces'
  • --option="Value with Spaces"

Boolean options​

Boolean options enable and disable settings. They have a true (enabled) and false (disabled) form:

  • Enabled (true): --option
  • Disabled (false): --no-option

The following examples compare Boolean option forms for REMOVE_API_KEYS.

  • True
  • False

--remove-api-keys is equivalent to setting PRIMEAGENT_REMOVE_API_KEYS=True in .env:


_10
uv run primeagent run --remove-api-keys

--no-remove-api-keys is equivalent to PRIMEAGENT_REMOVE_API_KEYS=False in .env:


_10
uv run primeagent run --no-remove-api-keys

In the following command references, default values for Booleans include both the CLI flag and the equivalent Boolean evaluation, such as "--option (true)" and "--no-option (false)".

Universal options​

The following options are available for all Primeagent CLI commands:

  • --version, -v: Show the version and exit.
  • --install-completion: Install auto-completion for the current shell.
  • --show-completion: Show the location of the auto-completion config file, if installed.
  • --help: Print information about command usage, options, and arguments.

CLI commands​

The following sections describe the available CLI commands and any additional options (beyond the universal options) available for each command.

primeagent​

Running the CLI without any arguments prints a list of available options and commands.

  • uv (recommended)
  • Direct

_10
uv run primeagent


_10
primeagent

primeagent api-key​

Creates a Primeagent API key.

You must be a superuser to create API keys with the CLI. For more information, see Primeagent API keys.

  • uv (recommended)
  • Direct

_10
uv run primeagent api-key


_10
primeagent api-key

Options​

OptionDefaultTypeDescription
--log-levelerrorStringThe logging level. One of debug, info, warning, error, or critical.

primeagent copy-db​

Copies the Primeagent database files from the cache directory to the current Primeagent installation directory, which is the directory containing __main__.py. You can find the copy target directory by running which primeagent.

The following files are copied if they exist in the cache directory:

  • primeagent.db: The main Primeagent database, stored in the user cache directory
  • primeagent-pre.db: The pre-release database, if it exists
  • uv (recommended)
  • Direct

_10
uv run primeagent copy-db


_10
primeagent copy-db

primeagent migration​

Manages Primeagent database schema changes using Alembic, a database migration tool for SQLAlchemy.

The migration command has two modes:

  • Test mode (default): Checks if migrations can be applied safely without actually running the migrations. Use this mode to previews the changes that would be made to the database schema before proceeding with the migrations.

  • Fix mode: Applies the migrations to update the database schema.

    warning

    primeagent migration --fix is a destructive operation that can delete data. Always run primeagent migration first to preview the changes.

  • uv (recommended)
  • Global
  1. Run test mode:

_10
uv run primeagent migration

  1. Preview the changes returned by the test to determine if it's safe to proceed with the migration.

  2. Run fix mode to apply the changes:


_10
uv run primeagent migration --fix

  1. Run test mode:

_10
primeagent migration

  1. Preview the changes returned by the test to determine if it's safe to proceed with the migration.

  2. Run fix mode to apply the changes:


_10
primeagent migration --fix

Start Primeagent with a specific .env file​

The --env-file option starts Primeagent using the configuration defined in the given .env file. Additional options appended to this command override the values in the .env file if there are duplicates.

If --env-file is omitted or doesn't include all required variables, Primeagent uses the default values for those variables.

  • uv (recommended)
  • Direct

_10
uv run primeagent run --env-file PATH/TO/PRIMEAGENT/.env


_10
primeagent run --env-file PATH/TO/PRIMEAGENT/.env

Start Primeagent in headless mode​

The --backend-only option starts Primeagent's backend service only. This headless mode has no frontend (visual editor), and you can only access the server programmatically with the Primeagent API and CLI.

  • uv (recommended)
  • Direct

_10
uv run primeagent run --backend-only


_10
primeagent run --backend-only

primeagent superuser​

Creates a superuser account with the given username and password.

  • uv (recommended)
  • Direct

_10
uv run primeagent superuser --username [NAME] --password [PASSWORD] [OPTIONS]


_10
primeagent superuser --username [NAME] --password [PASSWORD] [OPTIONS]

Options​

OptionDefaultTypeDescription
--log-levelerrorStringThe logging level. One of debug, info, warning, error, or critical.

For this command, --username and --password aren't optional, and they have no default value. The command fails if you don't provide these arguments. For more information, see PRIMEAGENT_SUPERUSER and PRIMEAGENT_SUPERUSER_PASSWORD.

Disable CLI superuser creation​

The primeagent superuser command is controlled by the PRIMEAGENT_ENABLE_SUPERUSER_CLI environment variable:

  • PRIMEAGENT_ENABLE_SUPERUSER_CLI=True (default): The primeagent superuser command is available, and superuser creation is unrestricted.
  • PRIMEAGENT_ENABLE_SUPERUSER_CLI=False (recommended): Disables the primeagent superuser command. For security reasons, this is recommended to prevent unauthorized superuser creation, especially in production environments.

To disable the primeagent superuser command, you must set PRIMEAGENT_ENABLE_SUPERUSER_CLI=False in your Primeagent .env file, and then start Primeagent with your .env file.
